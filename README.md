# ⚾ 숫자 야구 게임 코드 리뷰

## 📌 개요

본 프로젝트는 CSV 파일에서 정답을 읽어와 진행하는 숫자 야구 게임입니다.
기능 구현 이후 코드 구조 및 유지보수성을 기준으로 개선 포인트를 정리했습니다.

---

# 🔍 주요 개선 사항

## 1. 표준 헤더 명시 부족

### 문제

`remove`, `find_if`, `all_of`, `isspace`, `isdigit` 사용 시 필요한 헤더가 명시되어 있지 않음.

### 개선

```cpp
#include <algorithm>
#include <cctype>
```

### 정리

* 컴파일 환경 의존성 제거
* 코드 안정성 향상

---

## 2. 입력 처리 방식 (int 기반)

### 문제

정수 입력을 사용하여 `"023"` → `"23"`으로 변환됨

### 영향

* 선행 0 유지 불가
* 규칙 확장 시 구조 변경 필요

### 개선 방향

* 현재: 앞자리 0 금지 → 유지 가능
* 확장 고려 시: `string` 입력 방식 권장

### 한 줄 요약

> 자리 기반 게임에서는 문자열 입력이 더 적절함

---

## 3. 파일 2회 순회 구조

### 문제

* 전체 라인 수 카운트
* 파일 포인터 초기화 후 다시 탐색

### 영향

* 작은 파일: 문제 없음
* 큰 파일: 비효율 발생 가능

### 개선 방향

```cpp
vector<string> rows;
while (getline(file, line)) {
    rows.push_back(line);
}
```

* 한 번만 읽고 랜덤 선택

### 한 줄 요약

> 현재는 허용 가능하지만, 구조적으로는 개선 여지 있음

---

## 4. 판정 로직과 출력의 결합

### 문제

`Check()` 함수가 판정 + 출력 동시에 수행

### 영향

* 테스트 어려움
* 재사용성 낮음

### 개선 방향

```cpp
pair<int, int> Check(...);
```

→ 출력은 `Game()`에서 처리

### 한 줄 요약

> 로직과 UI는 분리하는 것이 바람직함

---

## 5. 함수 이름과 역할 불일치

### 문제

`setChance()`는 실제로 입력을 받지 않고 고정값 반환

### 개선 방향

* 기능 복구 또는
* 함수명 변경

### 한 줄 요약

> 함수 이름은 실제 책임을 정확히 반영해야 함

---

## 6. 불필요한 주석 코드 유지

### 문제

이전 구현 코드가 주석 상태로 남아 있음

### 영향

* 가독성 저하
* 유지보수 혼란

### 개선

* 제거하거나 Git 히스토리로 관리

### 한 줄 요약

> 과거 코드는 코드가 아니라 버전 관리에 남기는 것이 적절

---

## 7. 출력 메시지 개선 필요

### 문제

성공 시에도 정답을 다시 출력

### 개선 방향

* 성공: 축하 메시지
* 실패: 정답 공개

### 한 줄 요약

> 상태별 출력 분리가 사용자 경험에 유리

---

## 8. `using namespace std;` 사용

### 문제

전역 네임스페이스 오염 가능성

### 개선

```cpp
std::string
std::cout
```

### 한 줄 요약

> 실무에서는 명시적 네임스페이스 사용 권장

---

## 9. 입력 검증 로직 집중

### 문제

`Input()`에 검증 로직이 집중됨

### 개선 방향

```cpp
bool IsValidInput(const string& input);
```

### 한 줄 요약

> 입력과 검증은 분리하는 것이 확장성에 유리

---

## 10. 파일 처리 함수 책임 과다

### 문제

`GetData()`가 여러 역할 수행

* 파일 열기
* 라인 카운트
* 랜덤 선택
* CSV 파싱
* 문자열 정리
* 검증

### 개선 방향

```cpp
LoadRows()
PickRandomRow()
ParseCsv()
NormalizeAndCheck()
```

### 한 줄 요약

> 함수는 하나의 책임만 가지는 것이 유지보수에 유리
